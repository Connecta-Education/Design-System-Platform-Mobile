Can we build on PrimeReact without losing the Connecta look?

Short version for the dev team: yes — and without throwing away anything in this design system. PrimeReact themes through a layer of design tokens that compile to CSS variables. We point that layer at the variables already in colors_and_type.css, and PrimeReact's 90-plus components inherit the Connecta brand automatically. The CSS file stays the single source of truth.

PrimeReact is the engine. Connecta is the skin. They don't fight.

Use PrimeReact in styled mode with a custom preset whose tokens reference Connecta's existing CSS variables (var(--connecta-ink), var(--r-md), …). Nothing about Connecta's palette, type, radii or spacing changes — it just becomes reachable from PrimeReact. The handful of truly Connecta-native surfaces (chat bubbles, GoodTalk & Cosmo banners, role badges) stay hand-built, exactly as they are now.

01

How the pieces stack

PrimeReact (v10+) splits theming into three token tiers — primitive, semantic, component — that emit CSS custom properties prefixed --p-. We don't author a parallel palette there; we make those tiers defer to Connecta's variables. One source of truth, read top-to-bottom:

Your appJSX
PrimeReact components — <Button>, <DataTable>, <Dropdown>, <Calendar>… plus Connecta-native components for chat & AI surfaces.
PrimeReactstyled mode
Ships its own structural CSS, all of it driven by --p-* design tokens. No hard-coded colors in the components themselves.
Connecta presetdefinePreset
The bridge. Maps each --p-* token to a Connecta variable. ~60 lines, written once. This is the only new file engineering adds.
Connecta tokensthis repo
colors_and_type.css — the palette, type, radii, shadows, spacing you already approved. Unchanged. Source of truth.

Styled mode vs. unstyled — the recommendation

Styled mode + Connecta preset

Recommended

PrimeReact brings its component CSS; the preset recolors it through tokens. Least code, fastest to adopt, survives PrimeReact upgrades.

  • One ~60-line preset is the whole integration.
  • Connecta's look is opinionated but token-shaped (flat fills, hairlines, ink primary, fixed radii) — it maps cleanly.
  • Designers keep editing colors_and_type.css; PrimeReact follows.

Unstyled mode + passthrough

Escape hatch

PrimeReact ships zero CSS; you supply every class via the pt (passthrough) prop. Total control, much more code.

  • Reserve for the few components that must look exactly like a Connecta-native surface.
  • Overkill as the default — you'd be rebuilding what the preset gives you for free.
  • Both modes can coexist in one app.

03

The preset (drop-in)

This is the whole integration. One preset file plus a provider wrap. Because the tokens reference var(--connecta-*), the values live in colors_and_type.css — edit a colour there and every PrimeReact component updates with no rebuild of this file.

connecta-preset.js
// PrimeReact v10+ design-token theming (@primeuix/themes).
// Tokens point at Connecta's existing CSS variables — colors_and_type.css stays the source of truth.
import { definePreset } from '@primeuix/themes';
import Aura from '@primeuix/themes/aura'; // base structure only — every colour is overridden below

export const Connecta = definePreset(Aura, {
  semantic: {
    // — Primary = Ink (Connecta's single primary action) —
    primary: {
      color:         'var(--connecta-ink)',
      contrastColor: 'var(--connecta-paper)',
      hoverColor:    'var(--neutral-800)',
      activeColor:   'var(--connecta-ink)',
    },
    // — Shape & motion —
    borderRadius: {
      none: '0', xs: 'var(--r-xs)', sm: 'var(--r-sm)',
      md: 'var(--r-md)', lg: 'var(--r-lg)', xl: 'var(--r-xl)',
    },
    transitionDuration: 'var(--dur-base)',
    focusRing: {
      width: '2px', style: 'solid', offset: '2px',
      color: 'var(--connecta-ink)',
    },
    formField: {
      borderRadius:        'var(--r-sm)',
      focusBorderColor:    'var(--connecta-ink)',
      invalidBorderColor:  'var(--connecta-coral)',
      paddingX: '14px', paddingY: '11px',
    },
    colorScheme: {
      light: {
        surface: {
          0: 'var(--neutral-0)',   50: 'var(--neutral-50)',  100: 'var(--neutral-100)',
          200: 'var(--neutral-200)', 300: 'var(--neutral-300)', 400: 'var(--neutral-400)',
          500: 'var(--neutral-500)', 600: 'var(--neutral-600)', 700: 'var(--neutral-700)',
          800: 'var(--neutral-800)', 900: 'var(--neutral-900)', 950: 'var(--connecta-ink)',
        },
        text:    { color: 'var(--fg-1)', mutedColor: 'var(--fg-3)' },
        content: { background: 'var(--bg-card)', borderColor: 'var(--border-1)' },
      },
    },
  },
  components: {
    // Remap severities to Connecta's meaning-locked colours.
    // (Token names sit under components.<name>.colorScheme.light — generate
    //  against your installed version with the PrimeReact Theme Designer.)
    button: {
      colorScheme: {
        light: {
          danger:    { background: 'var(--connecta-coral)', color: 'var(--connecta-paper)' },
          secondary: { background: 'var(--neutral-100)',    color: 'var(--connecta-ink)'  },
          contrast:  { background: 'var(--connecta-ink)',    color: 'var(--connecta-paper)' },
        },
      },
    },
  },
});
main.jsx — wire it in once, light only
import { PrimeReactProvider } from 'primereact/api';
import { Connecta } from './connecta-preset';
import 'primereact/resources/primereact.min.css';
import './colors_and_type.css'; // ← the source of truth, loaded first

export default function Root() {
  return (
    <PrimeReactProvider value={{
      theme: { preset: Connecta, options: { darkModeSelector: false } },
    }}>
      <App />
    </PrimeReactProvider>
  );
}

Version note for the dev. The token-preset model above is PrimeReact v10+ (the @primeuix/themes engine). If you're on v9 or an older SCSS-theme build, the same result is reached differently — either override the theme's CSS variables directly, or use unstyled mode + passthrough. Confirm which major version you're pinning before wiring this up; the exact component-token names also shift between minors, so generate the components block against your installed version.

05

What stays custom

PrimeReact covers the generic 90% — forms, tables, overlays, navigation. The Connecta-native surfaces below have no PrimeReact equivalent and carry the brand's load-bearing identity. Keep them hand-built (they already are, in ui_kits/). They sit alongside PrimeReact, sharing the same tokens.

Chat bubbles

Self / friend / teacher / parent / Cosmo / GoodTalk bubble colours encode who is speaking. That's a Connecta invention — built from --bubble-* tokens, not a PrimeReact component.

GoodTalk & Cosmo banners

The in-flow intervention banner and Cosmo's Socratic panel have bespoke behaviour and voice. PrimeReact's Message can host simpler inline notes, but the AI surfaces stay native.

Role badges

The school-graph identity pills (Child · Teacher · Parent · School · Cosmo · GoodTalk). Trivial to render with a PrimeReact Tag + the --role-* tokens — shown in the gallery above.

Wordmark & app icon

Custom marks, never type set in DM Sans. Pure SVG assets in assets/ — outside any component library.

For the dev, in one line: add PrimeReact, drop in connecta-preset.js, wrap the app in PrimeReactProvider with darkModeSelector: false, keep loading colors_and_type.css. Use PrimeReact for forms / tables / overlays / nav; keep the chat & AI surfaces native. Nothing in this design system gets deleted or restyled.
