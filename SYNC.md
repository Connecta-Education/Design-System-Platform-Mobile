# Keeping the design system and this repo in sync

This repo and the **Claude Design tool** (`claude.ai/design`) are two sources of
truth that drift over time:

- The **design tool** regenerates the visual system and exports it as a
  *handoff bundle* (`…-handoff.tar.gz`) — HTML/CSS prototypes, tokens, the card
  manifest, and a runtime bundle.
- This **repo** additionally carries engineering decisions the design tool
  doesn't know about (the agent-facing instructions in `README.md`/`SKILL.md`,
  the lint config) and is deliberately leaner (no raw screenshots / scratch).

To stay in sync we use a fixed **ownership boundary** plus a one-command
importer (`scripts/sync-from-handoff.sh`), so each new export is a deterministic
merge instead of a manual reconciliation.

## Ownership boundary

| Class | Paths | On import |
|---|---|---|
| **Design-owned** (regenerated by the design tool) | `preview/`, `ui_kits/`, `review/`, `assets/`, `fonts/`, `colors_and_type.css`, `primereact-connecta.css`, `styles.css`, `_ds_manifest.json`, `_ds_bundle.js`, `.thumbnail`, `Connecta Design System - Guide.html`, `Connecta × PrimeReact.html` | **Overwritten** to match the handoff (dirs are mirrored, including deletions). |
| **Repo-owned** (engineering) | `README.md`, `SKILL.md`, `_adherence.oxlintrc.json`, `scripts/`, `SYNC.md`, `.github/` | **Never** touched by the importer. Edit by hand. |
| **Never-import** (scratch / pruned) | `uploads/*` (except the referenced product brief), `scraps/`, `*.prompt.md`, `Connecta DS - Overview & Cleanup.html`, stale bundles (`Connecta Chat App.html`, `Connecta PrimeReact Bridge.html`) | Skipped even if present in the handoff. |

> Why README/SKILL are repo-owned: they hold decisions made *here* that the
> design tool lags on — e.g. PrimeReact + custom-components as the component
> model, the Socratic "nn" badge mark, the navigation rule. A naive copy from a
> handoff would regress them. When the design intent genuinely changes, update
> these two files **by hand** during the import PR.

## Procedure

1. Download the latest handoff from the design tool (`…-handoff.tar.gz`).
2. From the repo root, on a fresh branch:
   ```bash
   git checkout -b sync/handoff-YYYY-MM-DD
   scripts/sync-from-handoff.sh ~/Downloads/Connecta\ Design\ System-handoff.tar.gz
   ```
   The script mirrors design-owned files, prints any **residual drift**
   (should be none), validates the manifest/bundle, and guards against legacy
   terms (`tamagui`/`cosmo`) sneaking back in.
3. `git status` / `git diff` — review what changed.
4. If the design changed something the docs describe (new component, renamed
   group, a new standing rule), update `README.md` and `SKILL.md` by hand.
5. Commit and open a PR into `main` (direct pushes to `main` are blocked by the
   environment; merge via PR).

## Closing the gap at the source (optional)

Bucket "repo-owned ahead of design" shrinks if you paste the repo's doc
decisions (PrimeReact instruction, nn-badge mark, navigation rule) back into the
design tool's own README/SKILL, so the next export already matches. Until then,
step 4 keeps them correct on the repo side.

## Current state

As of the last sync, the design source is fully in parity with the handoff; the
only differences are the intentional repo-owned and never-import items above.
